# Reference Writing Modes

Information-first writing supports the story instead of becoming the story. It
should be clear, durable, source-aware, and easy for future agents or humans to
use. Every entry needs a reader, a purpose, and a reason to exist.

## Fact Extraction

Convert chapters, scenes, or sessions into compressed story facts. Preserve what
changed: character state, relationships, timeline events, world rules, canon,
open questions, and settled decisions. Do not copy the prose; extract the state
future workers need.

Read before extracting:
- Source chapter, scene, or session notes
- Existing kb files for the same character, place, rule, or timeline
- Relevant vocab files

Write facts with source anchors. Update existing entries when facts change;
avoid duplicate parallel truth.

## Wiki / Reference Writing

Explain a story concept so future workers can use it: a location, faction,
magic rule, relationship, object, timeline, or recurring motif. Prioritize
mechanism and boundaries over pretty summary.

Read before writing:
- Source passages or decisions
- Existing related kb pages
- Vocab and naming conventions

A good reference page answers: what is it, how does it work, where is it shown,
what does it constrain, and what remains unknown?

## Vocabulary Writing

Record canonical terms, aliases, deprecated names, and boundaries. Vocabulary
writing prevents drift: two names for one thing, one name for two things, or a
term used outside its allowed meaning.

Each entry should include:
- canonical term
- definition or boundary
- aliases or deprecated forms when useful
- source or decision anchor
- related pages

## Decision Capture

Record settled creative choices when they will affect future work. Keep it
short: decision, reason, alternatives rejected, and what it changes downstream.
Do not promote live brainstorm options into decisions.

## Issue Logging

When a writing problem persists beyond the current pass, log it as an issue.
Name the symptom, why it matters to the reader, where it appears, attempts made,
and what kind of future pass should handle it.
